# Screenshots

Product screenshots for DailyDex GitHub README.

## Required Assets

- `overview-light.png` - Overview page in light mode
- `feed-cards.png` - Feed tab with card view
- `saved-board.png` - Saved workflow board
- `trends.png` - Trends page with charts
- `mobile-overview.png` - Overview on mobile viewport
- `demo-walkthrough.gif` - Animated walkthrough

## Viewport Recommendations

- Desktop screenshots: `1440px` width
- Mobile screenshots: `390px` width

## How to Capture

### Desktop

1. Open browser at `http://localhost:8888`
2. Set viewport to `1440px`
3. Take full-page screenshot or individual tab screenshots
4. Save as PNG with minimal compression

### Mobile

1. Open browser DevTools (F12)
2. Toggle device toolbar
3. Select iPhone 12 or similar (`390px`)
4. Navigate to Overview
5. Take screenshot

## When to Update Screenshots

- After major UI changes
- After rebrand or product name changes
- Before release candidates
- When tab structure changes visibly

## Checklist Before Committing

- [ ] All screenshots reflect current UI
- [ ] Product name shows "DailyDex" everywhere
- [ ] No visual regressions
- [ ] Mobile screenshot shows no horizontal overflow
- [ ] GIF animation loops smoothly
- [ ] File sizes are reasonable (PNG < 500KB, GIF < 2MB)
